# Recapture-5 — IOSHybridDirectMxReceiveTest on SM-T390 with permission shim

**Date:** 2026-05-18 15:35:54
**Goal:** Confirm the version-aware `GrantPermissionRule` shim added to
`IOSHybridDirectMxReceiveTest.kt` lets the instrumented test reach BLE
logic on SM-T390 (Android 9 / API 28) — previously blocked in setup.

## Devices

- **Android:** SM-T390 (5200f354f4fb277f), Android 9 / API 28.
- **iOS:** iPhone 13 (DairyPhoneDeaux, UDID `1780F216-…`) — intended emitter.

## Shim recap

`IOSHybridDirectMxReceiveTest.kt` line 94–113: version-aware rule.
- API ≥ 31 → `BLUETOOTH_ADVERTISE`, `BLUETOOTH_CONNECT`, `BLUETOOTH_SCAN`, `ACCESS_FINE_LOCATION`.
- API ≤ 30 → legacy `BLUETOOTH`, `BLUETOOTH_ADMIN`, `ACCESS_FINE_LOCATION`.

Before the shim: `GrantPermissionRule` requesting `BLUETOOTH_SCAN` etc.
threw at `@get:Rule` evaluation on API 28 (permissions don't exist), and
the test never reached `@Test`. The cheat sheet still flags the original
gotcha as historical.

## Run

- Fresh `app-debug.apk` + `app-debug-androidTest.apk` installed on T390.
- Test launched via `am instrument`, ran for 47.166 s (the full 45 s
  window + settle), produced an `Evidence bundle` line, then the
  expected content-assertion failure.
- iPhone emit was intended but the device was locked — `devicectl`
  returned `FBSOpenApplicationServiceErrorDomain error 1
  (RequestDenied)`. No emission happened during the window.

were possible before.
